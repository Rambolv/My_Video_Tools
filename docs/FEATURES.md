## 1. 视频去字幕水印功能

### 1.1 核心设置

#### 修复算法选择
- **类型**: 下拉选择框 + ? 帮助按钮
- **选项**: STTN-Auto / STTN-Det / LaMa / ProPainter / E2FGVI / OpenCV
- **功能**: 选择用于填充去除文字后区域的图像修复算法
- **E2FGVI 特殊处理**: 检测 GPU 显存，< 48GB 显示红色警告

#### 检测模型选择
- **类型**: 下拉选择框 + ? 帮助按钮
- **选项**: PP-OCRv4 Server/Mobile / PP-OCRv5 Server/Mobile / SAM2-Tiny/Small/Base/Large
- **功能**: 选择用于检测字幕文本区域的 AI 模型

#### 处理深度滑块
- **类型**: 水平滑块 (0-100) + 数值标签 + ? 帮助按钮
- **功能**: 连续调节所有模型参数的插值系数
- **映射关系**:
  ```
  depth=0  (极快):  mask膨胀=5px,   时间线=±1帧, STTN参考=5帧,  PP膨胀=4px
  depth=25 (轻度):  插值中间值
  depth=50 (标准):  mask膨胀=17px,  时间线=±5帧, STTN参考=12帧, PP膨胀=12px
  depth=75 (深度):  插值中间值
  depth=100(极致):  mask膨胀=30px,  时间线=±10帧, STTN参考=20帧, PP膨胀=20px
  ```
- **视觉反馈**: 滑块使用大圆形蓝色手柄 (18px)，悬停放大至 20px

### 1.2 性能设置

#### 硬件加速
- **类型**: SwitchButton 开关 + ? 帮助按钮
- **功能**: 启用/禁用 GPU 硬件加速（CUDA）

#### 最大并发任务数
- **类型**: 下拉选择框 (1-8) + ? 帮助按钮
- **特殊功能**: 根据 VRAM 估算自动标红超出显存的选项
- **颜色编码**: 红色 `#e81123` + ⚠️ 标记 = 可能超出显存

### 1.3 水印检测

- **类型**: 可折叠子功能区（▶/▼ 箭头 + 左边框线缩进）
- **包含**: 水印模板管理组件（截取/加载/预览/启用开关）
- **功能**: 检测变形 Logo/水印，支持模板匹配、颜色传播、强力清扫

### 1.4 显存估算与监控

- **类型**: 可折叠子功能区
- **包含**:
  - GPU 显存总量显示
  - 当前配置显存估算（修复+检测+深度系数，颜色编码）
  - VRAM 被动监控开关
  - 模型显存参考表（HTML 表格，真实采集值优先，OOM 标记 ☠️）

#### 颜色编码系统
```
≥ 95%: #e81123 (红色)   - 极危险
≥ 85%: #ff8c00 (橙色)   - 警告
≥ 70%: #ffd700 (黄色)   - 注意
< 70%: #16ab39 (绿色)   - 安全
```

### 1.5 操作按钮

| 按钮 | 图标 | 功能 |
|------|------|------|
| 打开 | 📂 | 选择视频/图片文件 |
| 开始 | ▶ | 开始处理任务 |
| 停止 | ⏹ | 终止当前处理 |
| ⚙ 高级设置 | ⚙ | 跳转高级设置页 |

## 2. 字幕提取功能

### 2.1 提取模式

| 模式 | 算法 | 输出格式 | 适用场景 |
|------|------|---------|---------|
| **按行提取** | Y 轴聚类 (3% 容差) → 行内 X 排序 | 行间换行 | 标准横向字幕 |
| **按列提取** | X 轴聚类 (3% 容差) → 列内 Y 排序 | 列间换行 | 竖向文字/表格 |
| **提取浮动字幕** | 欧氏距离聚类 (5% 容差) → 按 Y 排序 | 聚类合并 | 动态/浮动水印 |

### 2.2 智能去重

- **渐进文本检测**: 新文本完全包含上一结果的全部字符 → 覆盖为最终版本
- **时间窗口去重**: 相同文本在 0.5 秒内重复 → 跳过
- **文本标准化**: `_text_normalize()` 去空格/标点后对比

### 2.3 输出格式

| 目标 | 格式 | 内容 |
|------|------|------|
| 左侧 Tab「字幕文本」 | SRT | 序号 + 时间轴 + 文本 |
| 右侧预览框 | 纯文本 | 仅文本，条目间空行 |
| 「导出文本」按钮 | .txt | 纯文本（同预览框） |
| 「导出 SRT」按钮 | .srt | 标准字幕格式 |

### 2.4 联合校对

- **类型**: 可折叠子功能区
- **功能**: 3 个不同 OCR 模型依次提取后合并去重
- **约束**: 三个模型不可重复（自动互斥）
- **输出**: 合并后的 SRT + 纯文本

## 4. UI 设计规范

### 4.1 交互模式

| 元素 | 交互 |
|------|------|
| ? 帮助按钮 | 点击 → MessageBox 弹窗 / 悬停 0.6s → ToolTip |
| ▼/▶ 箭头 | 点击 → 展开/折叠内容，状态持久化到 config |
| 分割器手柄 | 拖拽 → 调整面板比例 |
| 滑块手柄 | 拖拽 → 实时更新数值和配置 |
| 标红选项 | 自动根据 VRAM 余量变化 |

### 4.2 视觉层次

```
功能区卡片 (CardWidget, border-radius: 8)
  ┃ SectionHeader (蓝色左边框 3px + 粗体)
  ┃ SettingRow (标签 80px + 控件 + ? 帮助)
  ┃ ▼ 子功能区 (蓝色粗体箭头 12px + 左边框线 2px)
  ┃   │ 缩进内容
```

### 4.3 响应式设计

- 主界面使用 `QSplitter`（水平 + 纵向）
- 右侧功能区使用 `QScrollArea`，内容过多时自动滚动
- 标签启用 `setWordWrap(True)`
- 按钮设置 `setMinimumHeight(34)` + `setMinimumWidth(80)`
- 文本区域启用 `setLineWrapMode(WidgetWidth)`

## 6. VRAM 监控系统

### 6.1 被动监控

- **原理**: 在用户正常工作流程中自动采样 GPU 显存峰值
- **触发**: `enableVramMonitoring` 配置项
- **频率**: 每 0.5 秒采样一次
- **存储**: `config/vram_records.json`
- **去重**: 相同配置不重复采集

### 6.2 数据应用

- 采集值覆盖 `_MODEL_VRAM_BASELINE_1080P` 内置基准
- 用于并发任务数显存标红
- OOM 记录标记为 ☠️

### 6.3 估算公式

```
单任务显存 = max(模型基准 × 分辨率系数 × 深度系数, 检测显存) × 1.10
多任务显存 = 首任务显存 + 次模型×0.5 + (并发数-1) × 首任务显存 × 0.40
超出判定 = total > GPU显存 × 0.90 (留 10% 余量)
```
