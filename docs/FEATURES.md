## 2. 字幕提取功能

### 2.1 提取模式

| 模式 | 算法 | 输出格式 | 适用场景 |
|------|------|---------|---------|
| **按行提取** | Y 轴聚类 (3% 容差) → 行内 X 排序 | 行间换行 | 标准横向字幕 |
| **按列提取** | X 轴聚类 (3% 容差) → 列内 Y 排序 | 列间换行 | 竖向文字/表格 |
| **提取浮动字幕** | 欧氏距离聚类 (5% 容差) → 按 Y 排序 | 聚类合并 | 动态/浮动水印 |

### 2.2 智能去重

- **渐进文本检测**: 新文本完全包含上一结果的全部字符 → 覆盖为最终版本
- **时间窗口去重**: 相同文本在 0.5 秒内重复 → 跳过
- **文本标准化**: `_text_normalize()` 去空格/标点后对比

### 2.3 输出格式

| 目标 | 格式 | 内容 |
|------|------|------|
| 左侧 Tab「字幕文本」 | SRT | 序号 + 时间轴 + 文本 |
| 右侧预览框 | 纯文本 | 仅文本，条目间空行 |
| 「导出文本」按钮 | .txt | 纯文本（同预览框） |
| 「导出 SRT」按钮 | .srt | 标准字幕格式 |

### 2.4 联合校对

- **类型**: 可折叠子功能区
- **功能**: 3 个不同 OCR 模型依次提取后合并去重
- **约束**: 三个模型不可重复（自动互斥）
- **输出**: 合并后的 SRT + 纯文本

## 6. VRAM 监控系统

### 6.1 被动监控

- **原理**: 在用户正常工作流程中自动采样 GPU 显存峰值
- **触发**: `enableVramMonitoring` 配置项
- **频率**: 每 0.5 秒采样一次
- **存储**: `config/vram_records.json`
- **去重**: 相同配置不重复采集

### 6.2 数据应用

- 采集值覆盖 `_MODEL_VRAM_BASELINE_1080P` 内置基准
- 用于并发任务数显存标红
- OOM 记录标记为 ☠️

### 6.3 估算公式

```
单任务显存 = max(模型基准 × 分辨率系数 × 深度系数, 检测显存) × 1.10
多任务显存 = 首任务显存 + 次模型×0.5 + (并发数-1) × 首任务显存 × 0.40
超出判定 = total > GPU显存 × 0.90 (留 10% 余量)
```

## 7. 视频增强（超分辨率 & 帧插值）

### 7.1 概述

本模块为 VSR 魔改版新增的视频质量增强功能，包含**超分辨率（Super-Resolution）** 和 **帧插值（Frame Interpolation）** 两个子模块，可分别使用或组合使用（先超分后插帧）。

### 7.2 超分辨率（Real-ESRGAN）

| 项目 | 说明 |
|------|------|
| **算法** | Real-ESRGAN（BSRGAN 改进版，BSD 3-Clause 许可） |
| **Python CUDA 后端** | 通过 pip 安装 `realesrgan` 包，模型自动下载至 `resources/weights/` |
| **ncnn-Vulkan 后端** | 通过 `realesrgan-ncnn-vulkan.exe` 调用，⚠️ 模型文件托管于 GitHub LFS，国内网络受限，自动回退 Python |

**状态指示**：
- ✅ **Python CUDA**: 可用（✅ 2026-06-23 实测 75帧 640→2560 耗时 10s）
- ⚠️ **ncnn-Vulkan**: 模型文件（.bin/.param）托管在 GitHub LFS，国内下载失败时自动回退 Python

**可选模型**：
```
realesr-general-x4v3         # ⚠️ 默认，可能需下载
RealESRGAN_x4plus            # ✅ 通用 4x 超分
RealESRGAN_x4plus_anime_6B   # 动漫优化版
RealESRGAN_x2plus            # 2x 超分
```

### 7.3 waifu2x 动漫超分

| 项目 | 说明 |
|------|------|
| **引擎** | waifu2x-ncnn-vulkan（MIT 许可，v20250915） |
| **后端** | ncnn-Vulkan（仅此一种） |
| **模型架构** | `cunet`（通用动漫）/ `upconv_anime`（轻量动漫） |
| **降噪级别** | 0-3 级 |
| **缩放比例** | 1x-10x（自定义步长 0.1） |
| **多线程** | 可配置，默认 `1:2:2`（处理:缩放:去噪） |

**状态**：✅ 可用

### 7.4 帧插值（RIFE）

| 项目 | 说明 |
|------|------|
| **算法** | RIFE（Real-Time Intermediate Flow Estimation）v3.x |
| **模型来源** | Flowframes RIFE39 模型（flownet.pkl，27MB） |
| **插值倍数** | 2x / 3x / 4x / 8x |

#### Python CUDA 后端

| 方面 | 说明 |
|------|------|
| **原理** | PyTorch 加载 flownet.pkl，光流法插值中间帧 |
| **状态** | ✅ **已修复（2026-06-23）** — 改用 `importlib` 直接加载模块，绕过 spawn 子进程包导入问题 |
| **依赖** | PyTorch（CUDA），嵌入版 Python 已预装 |
| **模型路径** | `resources/models/rife/flownet.pkl` |

**历史问题**：
- ⚠️ `No module named 'model.RIFE_HDv3'` → ✅ 已修复（改为 `from .model.RIFE_HDv3`）
  - 根因：QPT 环境下 sys.path 不含子包路径，绝对导入失败
- ⚠️ **子进程导入偶发失败** → ✅ 已修复（改用 `importlib.util.spec_from_file_location` 直接加载，不依赖 sys.path）
- ✅ **子进程测试通过**：`multiprocessing.spawn` 子进程中 `_ensure_rife()=True`, `FI available=True`

#### ncnn-Vulkan 后端

| 方面 | 说明 |
|------|------|
| **引擎** | `rife-ncnn-vulkan.exe`（来自 Flowframes 旧版，4.1MB） |
| **工作模式** | 配对模式（`-0 f0 -1 f1 -o out`），逐对处理 |
| **状态** | ⚠️ **配对模式可用但慢** — 每对帧启动一次 exe，N 帧需要 N-1 次启动 |

**已知问题**：
- ❌ **目录模式已废弃**：旧版 exe 处理 rife-v3 模型时输出混乱（4 帧输入 → 8 帧输出，多半损坏）

- ⚠️ **模型兼容性**：旧版 exe 不支持 rife-v4.6 的 `Eltwise` 层，需使用 rife-v3.1 模型

### 7.5 增强管道工作流

```
输入视频 → [可选] 超分辨率（Real-ESRGAN / waifu2x）
          → [可选] 帧插值（RIFE）
          → 输出视频（保留原始音频）
```

管道由 `VideoSuperResolution` 和 `VideoFrameInterpolation` 两个类串联：
1. 超分阶段：提取所有帧 → 逐帧超分 → 编码为临时视频
2. 插帧阶段：提取超分后帧 → RIFE 插值 → 编码为最终视频 + 原始音频

**实测性能 (2026-06-23, RTX 4090)**：
| 操作 | 输入 | 输出 | 耗时 |
|------|------|------|------|
| Real-ESRGAN 4x SR | 75帧 640×360 | 75帧 2560×1440 | 10.1s |
| RIFE 2x FI | 75帧 15fps | 149帧 30fps | 2.5s |
| 组合管道 (SR→FI) | 75帧 640×360 15fps | 149帧 2560×1440 30fps | 182.7s (含模型下载)

## 9. AI 音频工作室 (Post-v1.4.0)

### 9.1 概述

整合了 VoxCPM2（语音合成）和 ACE-Step 1.5（音乐生成）两大 AI 引擎，提供统一的 Gradio WebUI 入口。

### 9.2 VoxCPM2 语音引擎

| 功能 | 说明 |
|------|------|
| **语音合成 (TTS)** | 输入文本生成语音，支持多种音色预设 |
| **音色克隆 (Clone Voice)** | 从参考音频克隆音色，生成新语音 |
| **声音设计 (Design Voice)** | 文本描述生成定制声音 |
| **声音转换 (Convert Voice)** | 输入音频转换为目标音色 |

**技术细节**：
- **架构**：AudioVAE V2 + LocEnc → TSLM → RALM → LocDiT
- **输出格式**：48kHz 采样率
- **模型权重**：~4.7GB
- **硬件**：RTX 4090 24GB 可运行

### 9.3 ACE-Step 1.5 音乐引擎

| 功能 | 说明 |
|------|------|
| **文生音乐 (Text to Music)** | 文本描述生成完整音乐 |
| **歌词生曲 (Lyrics to Song)** | 输入歌词生成配乐演唱 |
| **续写 (Continue)** | 从指定时间点续写音乐 |
| **重绘 (Repaint)** | 替换指定时间段的音乐内容 |
| **翻唱 (Cover Song)** | 输入音频 + 歌词生成翻唱版本 |
| **音源分离 (Separate)** | 分离人声/伴奏/鼓/贝斯/其他 |
| **LoRA 训练** | 自定义音色风格微调 |

**技术细节**：
- **架构**：LM 规划器 (CoT) + DiT 扩散
- **Turbo 模式**：4-step 快速生成
- **模型权重总量**：~9.6GB（DiT 4.5GB + LM 3.5GB + VAE 322MB + TextEnc 1.1GB）

## 10. GPU 检测式让步机制 (Post-v1.4.0)

### 10.1 概述

解决字幕去除过程中 GPU 持续满载导致桌面卡顿（DWM 饥饿）的问题。**不硬性等待，而是智能检测并微让步**。

### 10.2 工作原理

```
_gpu_yield_if_busy()  — 每轮进度更新时调用
                         │
                    ┌────┴────┐
                    │ 启用条件 │ ← _gpu_yield_enabled = True（仅模型处理中）
                    └────┬────┘
                         │
                    ┌────┴────┐
                    │ 时间阈值 │ ← 渐进式检测间隔
                    └────┬────┘
                         │
                    ┌────┴──────────────┐
                    │ torch.cuda.synchronize() │ ← 耗时估算 GPU 繁忙度
                    └────┬──────────────┘
                         │
              ┌──────────┴──────────┐
              │                     │
         sync < 1ms            sync > 10ms
         (GPU 空闲)            (GPU 繁忙)
              │                     │
        间隔→10s              间隔→0.5s
                              让步 1ms
```

### 10.3 参数说明

| 参数 | 空闲态 | 繁忙态 |
|------|--------|--------|
| 检测间隔 | 10s（指数增长） | 0.5s（指数衰减） |
| sync 阈值 | < 1ms | > 10ms |
| yield 动作 | 不动作 | `time.sleep(0.001)` |

### 10.4 生命周期

| 阶段 | `_gpu_yield_enabled` | 行为 |
|------|---------------------|------|
| 空闲（无处理） | `False` | 不检测、不动作、零开销 |
| 模型加载 → 处理中 | `True` | 每次进度更新时检测 |
| 处理完成 → 模型卸载 | `False` | 恢复不动作 |
